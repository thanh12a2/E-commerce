Dedicated code zone

The gateway now has a clear home in code. Definitions, registry building, and public views are split so the API gateway can evolve without burying route knowledge inside unrelated customer logic.

Downstream topology

Every browser-facing flow still terminates at user_service. The gateway forwards catalog, commerce, and AI work to the right service while keeping public website contracts stable.

Gateway route workspace

Routes are grouped by what they do, not by file. This makes it easier to inspect the real public contract of the gateway and the upstream services each flow depends on.

This workspace is generated from local gateway metadata, so you can inspect the public contract at /gateway/ and pull the same source of truth as JSON from /gateway/apis/.
